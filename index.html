The Pulpit Music Blog

    See the Code Go To Site

A friend of mine came to me asking for a custom blog that he, as a writer could edit fairly easily, but he did not want to use Wordpress. So, after doing a small amount of research I stumbled upon Jekyll.

Jekyll uses a combination of Ruby, HTML, and Liquid. Jekyll builds static HTML files from a structure and markdown files with the blogger's writing.

The blogger requested a system to categorize his posts, which is not usually done with Jekyll blogs, but after creating only a couple custom pages I was able to create the site that exists today.

Please feel free to check out the site, but remember, I am not the author.

Wynn Casino Boston Construction

At my summer internship for a web development company in Boston, Getfused, I worked on many websites as a Front-End Developer. One of those sites was for Wynn Casino. My role on the Getfused team was to wrap each site, which means that I use CSS in order to fully style the site based on comps given to me by a developer.

On this specific site, there were a few unique problems that I had to solve. First, the client asked for a fully responsive image map, so after doing some research I found that implementing JavaScript along with set coordinates in the html was the easiest way to do this. The photo gallery was a task that required JavaScript and extensive CSS styling in order to account for all possible scenarios (screen sizes, caption sizes, etc.).

I will add a link to the page when the site has gone live, but for now all that I can share is some teaser photos and my experience with the site.

My Portfolio (This Site)

    See the Code

I made my portfolio almost completely from scratch. The only aspect that I did not personally create was the bootsrap modals I used to showcase my work. I thought it would be a good idea to get some experience with implementing Bootstrap into a project because it is a very useful tool.

While making this page, I did not want to repeatedly go to Adobe Photoshop in order to create mockups of each of my projects, which is the standard procedure for creating mockups. Instead, I used standard screenshots from Google Chrome and positioned them with CSS, so that when I do another project, I can take the screenshots and set the directory for each image in the mockup and they will automatically resize and position themselves. Skipjack's

    Go To Site

Skipjack’s was my third site at my Getfused internship. This site required CSS styling that replicated the designer’s comps. It’s unique quality from the rest of my projects is its header. The header shrinks when you scroll down the page (just like the header on this site) and it also has a tag in the middle of the header which required special attention.

Also, the expanded header tag has a semi-circle at the bottom of it, so instead of using an image that would slow down the site, I used a CSS shape. The shape is actually a cone, which is partially hid behind the rectangular section of the tag.

I will add a link to the page when the site has gone live, but for now all that I can share is some teaser photos and my experience with the site.
